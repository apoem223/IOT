## 1 项目应用场景说明：
本项目是基于烟雾报警装置而构建的一款串口通信 + socket +MQTT通信的家庭防火报警系统，通过终端检测烟雾浓度，温湿度，光强大小，协调器接收数据，并将采集到的烟雾数据发送给上位机，当数值超过阈值大小时，上位机启动LED变色，并播放报警音乐，再通过socket通信，将警报同步给手机端。同时，云平台实时接收数据，可适用于家庭厨房、卧室、阳台等区域，通过对报警系统的研究和设计，可以更好的帮助人们实现智能报警，避免危害情况的发生。

## 2 系统功能概述及框架：
（1）系统功能概述
实时监测功能：终端节点通过烟雾，温湿度，光强传感器实时采集环境数据。
无线数据传输：终端节点将采集到的数据通过ZigBee协议发送至协调器。
上位机处理与显示：协调器通过串口将数据上传至PC上位机，上位机软件实时显示烟雾浓度，温湿度，光强数据，并把数据上报给云平台。
智能报警机制：当烟雾浓度超过预设阈值时，PC端自动触发声光报警（LED变色、播放报警音乐，继电器闭合）。
多终端联动：通过Socket通信将报警状态同步至手机安卓客户端，实现远程监控与报警提示。
手动/自动模式切换：用户可根据需要切换系统工作模式，灵活控制报警响应逻辑。

系统采用三层分布式架构：
1.感知层：由搭载CC2530芯片的终端节点与烟雾传感器、温湿度传感器、光强传感器组成负责环境数据的采集与初步处理。
2.网络层：协调器负责组建ZigBee网络，接收终端节点数据，并通过串口转发至上位机。
3.应用层：PC上位机负责数据解析、阈值判断、报警控制，并通过Socket服务与安卓客户端进行通信，实现跨平台报警同步。同时，云平台收集数据。

## 3 程序运行调试状况
打开串口，终端开始周期性地采集温度,烟雾，光强数据。同时，打开服务器，安卓端与PC端连接在同一IP地址上。协调器把从终端采集到的数据发给上位机，上位机进行数据判断，若烟雾浓度大于阈值，触发报警指示，同步给安卓端，安卓端小灯闪烁。并且把数据上报给云平台。
